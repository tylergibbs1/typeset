# Rendering Engines

Typeset supports two rendering engines. Set the `engine` field on any render request to choose.

## HTML Engine (default)

Uses Playwright + headless Chromium. Supports any HTML/CSS template with Handlebars-style data injection.

```json
{
  "engine": "html",
  "template": "<html><body><h1>{{title}}</h1></body></html>",
  "data": { "title": "My Report" }
}
```

**Template features:**
- `{{variable}}` — simple substitution (HTML-escaped)
- `{{#each items}}...{{/each}}` — array iteration
- `{{#if condition}}...{{/if}}` — conditional rendering

**AI Layout Engine:** When `smartLayout: true` (the default), GPT-5.4 analyzes the template and determines optimal page breaks, table splitting, and orphan/widow prevention.

## Typst Engine

Uses the Typst compiler (Rust binary). Templates are written in Typst markup. Data is loaded from a JSON file.

```json
{
  "engine": "typst",
  "template": "#let data = json(\"data.json\")\n= #data.title",
  "data": { "title": "My Report" }
}
```

<Callout type="info" title="Auto-detection">
If you don't set `engine`, Typeset detects it automatically. Templates starting with `#set`, `#let`, or `#import` are treated as Typst. Everything else is HTML.
</Callout>

## Comparison

| Feature | HTML | Typst |
|---------|------|-------|
| Template language | HTML/CSS | Typst markup |
| Table pagination | AI-assisted | Native |
| PDF size | ~90KB | ~30KB |
| Binary dependency | Chromium (~400MB) | Typst (~30MB) |
| Render time | ~500ms | ~500ms |
| AI layout | Yes | Not needed |
| PDF/UA accessibility | No | Yes |
