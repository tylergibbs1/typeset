# Typeset

Typeset is a **document computation engine** with a closed-loop architecture:

```
input data → template → render → verify → extract → loop
```

Every document generation is a **pipeline run**: a deterministic, auditable, verifiable unit of work.

<Callout type="info" title="Core Guarantee">
The same input + template always produces the same output. The output is verified against the input. The verification result is stored alongside the output.
</Callout>

## The Three Primitives

| Primitive | Input | Output |
|-----------|-------|--------|
| **Render** | data + template | document (PDF) |
| **Verify** | document | correctness score + issues |
| **Extract** | document | structured data |

These compose into pipelines. Extract from a messy PDF, render into a clean one, verify the output matches. That's one pipeline run with full lineage.

## Two Rendering Engines

<Tabs items={['HTML (Playwright)', 'Typst']}>
<Tab value="HTML (Playwright)">
The default engine. Renders any HTML/CSS template via headless Chromium.

```typescript
const result = await typeset.render({
  template: '<h1>{{title}}</h1><p>{{body}}</p>',
  data: { title: 'Hello', body: 'World' },
  engine: 'html',
});
```

Best for: web-native templates, complex CSS layouts, existing HTML content.
</Tab>
<Tab value="Typst">
Native document markup compiled to PDF. Tables break across pages automatically. No browser needed.

```typescript
const result = await typeset.render({
  engine: 'typst',
  template: '#let data = json("data.json")\n= #data.title\n#data.body',
  data: { title: 'Hello', body: 'World' },
});
```

Best for: long documents, large tables, accessibility (PDF/UA), minimal infrastructure.
</Tab>
</Tabs>
